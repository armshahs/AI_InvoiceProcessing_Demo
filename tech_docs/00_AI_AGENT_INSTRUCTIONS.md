# AI Agent Instructions

## Objective

Build a polished interactive HTML MVP for an AI-powered Purchase Order Ingestion platform using the specifications in this repository.

## Working Rules

1. Read **all markdown files** before writing code.
2. Treat `01_Global_Design_System.md` as the single source of truth for styling.
3. Treat every screen specification under `/screens` as mandatory.
4. Build reusable components first.
5. Use mock JSON/static data only.
6. No backend, authentication or database.
7. Simulate AI processing with animations.
8. Ensure consistent navigation and layouts across all screens.
9. Build responsive desktop-first pages.
10. Do not skip screens because they appear similar.

## Recommended Build Order

1. Layout shell
2. Global styles
3. Reusable components
4. Dashboard
5. Inbox
6. AI Processing
7. Business Validation
8. Maker Review
9. Checker Approval
10. ERP Export
11. Audit Trail
12. AI Copilot
13. Final visual polish

## Acceptance Criteria

- Every documented screen exists.
- Navigation works.
- Sample data is realistic.
- Animations are smooth.
- Components are reused.
- Enterprise-quality visual design.
